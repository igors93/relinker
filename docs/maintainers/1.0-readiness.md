# 1.0 readiness

This document defines readiness criteria for a future `1.0` release. It does not
declare Relinker ready for `1.0`.

## Automated requirements

- [x] Public root API snapshot exists.
- [x] Documented module API snapshot exists.
- [x] Behavioral contract suite covers sync, async, decorators, and context managers.
- [x] Coverage floor is enforced.
- [x] Built wheel is installed and behavior-smoke-tested.
- [x] Public typing examples are checked by mypy.
- [x] Documentation links and examples are tested.

## Release requirements

- [ ] Release `0.9.0`.
- [ ] Complete at least one full feedback cycle on the `0.9.x` series.
- [ ] Resolve all known critical and high-severity defects.
- [ ] Confirm no incompatible API changes are planned for `1.0`.
- [ ] Review the complete changelog and migration guidance.

## External validation

- [ ] Validate the `0.9.x` series in at least one real application or representative integration.
- [ ] Record feedback about sync, async, context-manager, and Retry Budget usage.
- [ ] Confirm cancellation and shutdown behavior in a real async runtime.

These items must not be checked by automation alone.

## API review

The final review should cover:

- root exports in `relinker.__all__`;
- documented module exports in `relinker.context.__all__`;
- retry policy builder methods;
- exceptions and control-flow helpers;
- HTTP helpers and presets;
- statistics, diagnostics, simulation, and result objects.

## Known non-goals

- distributed retry budgets
- circuit breaker
- hard interruption of running functions
- task queue
- scheduler

## Decision

`1.0` should be published only after the relevant release requirements and
external validation items are complete. Automated contracts are necessary but
not sufficient for that decision.
