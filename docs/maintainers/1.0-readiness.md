# 1.0 release record

This document records the technical evidence used to prepare Relinker 1.0.0.
It does not claim external adoption, user count, or production history that is
not recorded in the repository.

## Automated evidence

The following items were verified by automated tooling before this release:

- [x] Public root API snapshot exists and passes (`relinker.__all__`).
- [x] Documented module API snapshot exists and passes (`relinker.context.__all__`).
- [x] Behavioral contract suite covers sync, async, decorators, and context managers.
- [x] Coverage floor (85%) is enforced and met.
- [x] Built wheel is installed and behavior-smoke-tested in an isolated environment.
- [x] Public typing examples are checked by mypy strict.
- [x] Documentation links and examples are tested.
- [x] Ruff format and lint pass globally.
- [x] mypy strict passes on `src` and `tests/typing`.
- [x] `python -m build` produces wheel and sdist without errors.
- [x] Twine check passes with `--strict`.
- [x] CI matrix covers Python 3.10, 3.11, 3.12, and 3.13.

## Maintainer release decision

The following decisions were made by the maintainer for this release:

- The version is being stabilized as `1.0.0`.
- No incompatible changes to the current public API surface are planned.
- The compatibility and deprecation policy documented in
  `docs/reference/compatibility.md` takes effect from this release.
- Future breaking changes require a major version increment.

## Known scope boundaries

- `RetryBudget` is process-local and in-memory. It does not coordinate across
  processes or machines.
- Relinker is not a circuit breaker.
- `max_time()` does not interrupt a user function that is already running.
- Relinker is not a task queue.
- Relinker is not a scheduler.
- Relinker is not a distributed rate limiter.

## Decision

The repository is technically prepared for version 1.0.0. The release decision
is owned by the maintainer. This record documents automated evidence and known
scope boundaries without claiming unrecorded external adoption.
